# Self-Learning Skill - 自动学习与经验总结

## 何时调用

**⚠️ 重要：以下情况必须调用此 skill：**

- **任务完成时** - 无论成功或失败，都必须总结经验
- **遇到重要问题时** - 发现新的问题、解决方案或最佳实践时
- **学习到新知识时** - 获得新的技术知识、工具使用方法等
- **任务失败后** - 分析失败原因，记录避免方法
- **发现可复用模式时** - 发现可复用的代码模式、解决方案等
- **调取以前的经验时** - 当需要回顾以前的成功经验、失败教训、关键步骤等时

## 执行流程

每次完成任务时，按以下步骤执行：

1. **任务完成后立即总结** - 分析任务的成功或失败
2. **提取关键信息** - 提取成功经验、失败原因、关键步骤等
3. **存储到 memory** - 将经验存储到 memory 文件夹
4. **更新 SKILL 文件** - 将特别重要的经验直接更新到 SKILL.md
5. **索引经验** - 为经验添加标签，便于后续检索

## Memory 文件夹结构

```
memory/
├── experiences/           # 经验记录
│   ├── success/          # 成功经验
│   │   ├── {timestamp}_{short_summary}.md
│   │   └── ...
│   ├── failures/         # 失败教训
│   │   ├── {timestamp}_{short_summary}.md
│   │   └── ...
│   └── patterns/         # 可复用模式
│       ├── {timestamp}_{short_summary}.md
│       └── ...
└── index.md             # 经验索引
```

## 经验记录格式

### 记录经验调用脚本

脚本路径：`./scripts/saveExperience.js`
参数说明：

```json
{
  "type": "success|failure|pattern",
  "timestamp": "2023-10-15T10:30:00",
  "summary": "任务类型 - 成功经验",
  "content": "详细的成功经验记录"
}
```

## 读取和应用经验

1. **获取到经验文件列表** - 读取`./memory/index.md`文件获取到已有的经验文件列表
2. **获取到经验文件详情** - 结合需要获取的经验和已有的经验文件列表，读取对应的经验文件详情

## 最佳实践

1. **及时记录** - 任务完成后立即记录，避免遗漏
2. **详细准确** - 记录要详细、准确，便于后续参考
3. **结构清晰** - 使用统一的格式，便于阅读和检索
4. **标签管理** - 合理使用标签，提高检索效率
5. **定期整理** - 定期整理和归档旧经验
6. **持续改进** - 根据实际使用情况，改进记录格式和内容
